🚀 Phase 2 – Translating SQL Concepts to PySpark (Joins & Aggregations)
📌 Overview

In this phase, I worked on implementing core SQL operations—specifically joins and aggregations—using PySpark. The tasks were based on practical, real-world datasets to strengthen applied understanding.

💡 Methodology

My approach was to first solve each problem using SQL queries and then recreate the same logic using PySpark DataFrame operations. This comparison made it easier to see how both frameworks handle data transformations.

✅ Key Skills Developed

Using join() to merge multiple DataFrames
Applying groupBy() along with aggregation functions like sum() and avg()
Filtering aggregated results using filter()
Sorting outputs with orderBy()\

🧠 Conceptual Understanding

Through this process, I gained clarity on how data from different sources can be combined effectively and how aggregation functions help in extracting meaningful insights such as total expenditure and average metrics.

⚠️ Challenges Encountered

Defining correct join conditions
Differentiating between inner joins and left joins
Performing multiple aggregations within a single groupBy() operation

🔧 Problem-Solving Approach

Practiced SQL and PySpark implementations side-by-side
Divided complex problems into smaller, manageable steps
Validated results after each stage to ensure accuracy
🛠 Tools & Platforms
DB-Fiddle for executing SQL queries
Spark Playground for experimenting with PySpark

📊 Result

By the end of this phase, I became confident in applying joins and aggregations in PySpark and developed a clear understanding of how traditional SQL logic translates into PySpark transformations.
